## 一、项目概述

### 1.1 产品功能

Ultra-TM02型超低温温度测量模块主要功能：

| 功能 | 描述 |
|------|------|
| **温度测量** | 测量二极管探头的节电压，转换为温度值 |
| **温度显示** | 3.5寸LCD显示温度值，分辨率0.001℃ |
| **4-20mA输出** | 将温度值转换为4-20mA电流信号输出 |
| **USB通讯** | 与上位机软件通讯，参数设置 |
| **分度表下载** | 支持用户自定义探头分度表 |
| **参数存储** | 参数存储到Flash，掉电不丢失 |

### 1.2 主要技术指标

| 参数 | 规格 |
|------|------|
| 温度测量范围 | 1.4K ~ 500K (-271.75℃ ~ 226.85℃) |
| 显示分辨率 | 0.001℃ |
| 最大允许误差 | 低温段 ±1℃，高温段 ±2℃ |
| ADC精度 | 24位 |
| 内部电流源精度 | ≤0.05% |
| 数据更新率 | 最高1次/秒 |
| 探头类型 | 二极管式（4线差分） |
| 激励电流 | 10μA / 17μA 可选 |
| 输出形式 | 4-20mA（两线制） |

## 四、功能模块设计

### 4.1 温度测量模块

#### 4.1.1 功能描述

- 控制DAC输出激励电流（10μA/17μA）
- 采集二极管正向压降（ADC）
- 根据分度表查表计算温度值
- 滤波处理，提高测量稳定性

#### 4.1.2 测量流程

```
┌─────────────┐
│  开始测量   │
└──────┬──────┘
       ↓
┌──────────────┐
│ DAC输出电流  │  → 10μA 或 17μA
└──────┬───────┘
       ↓
┌──────────────┐
│ 等待稳定     │  → 延时10ms
└──────┬───────┘
       ↓
┌──────────────┐
│ ADC采集电压  │  → 24位ADC
└──────┬───────┘
       ↓
┌──────────────┐
│ 数字滤波     │  → 中值滤波 + 均值滤波
└──────┬───────┘
       ↓
┌──────────────┐
│ 查分度表     │  → 4871点曲线
└──────┬───────┘
       ↓
┌──────────────┐
│ 计算温度     │  → 线性插值
└──────┬───────┘
       ↓
┌──────────────┐
│ 更新显示     │
└──────────────┘
```

### 4.4 4-20mA输出模块

#### 4.4.1 功能描述

- 将温度值线性映射到4-20mA
- 用户可设置4mA和20mA对应的温度点
- 通过隔离DAC控制V/I转换电路

#### 4.4.2 转换公式

```
I_out = 4 + (T_current - T_4mA) / (T_20mA - T_4mA) × 16

其中：
- I_out: 输出电流 (mA)
- T_current: 当前温度 (℃)
- T_4mA: 4mA对应温度点 (℃)
- T_20mA: 20mA对应温度点 (℃)
```

#### 4.4.3 代码实现

```c
// 4-20mA输出配置
typedef struct {
    float temp_4mA;         // 4mA对应温度
    float temp_20mA;        // 20mA对应温度
    float current_mA;       // 当前输出电流
} Output420mA_t;

// 计算输出电流
float APP_Output_CalcCurrent(float temperature)
{
    float current;
    Output420mA_t *cfg = &g_output_cfg;
    
    // 线性插值
    current = 4.0f + (temperature - cfg->temp_4mA) / 
              (cfg->temp_20mA - cfg->temp_4mA) * 16.0f;
    
    // 限幅
    if (current < 4.0f) current = 4.0f;
    if (current > 20.0f) current = 20.0f;
    
    return current;
}
```

| 命令码 | 名称 | 描述 |

| 0x03 | GET_VOLTAGE | 获取电压值 |
| 0x10 | SET_CURRENT_SRC | 设置电流源 |
| 0x11 | SET_CURRENT_ADJ | 设置电流调整值 |
| 0x20 | SET_4MA_TEMP | 设置4mA温度点 |
| 0x21 | SET_20MA_TEMP | 设置20mA温度点 |

| 0x31 | STOP_ACQ | 停止采集 |
| 0x40 | LOAD_TABLE | 加载分度表 |
| 0x50 | SAVE_PARAM | 存储参数 |

### 5.3 命令详细定义

#### 5.3.1 获取设备ID (0x01)

**请求：**
```
AA 01 00 [CRC] 55
```

**响应：**
```
AA 01 10 [16字节设备ID] [CRC] 55
```

#### 5.3.2 获取温度值 (0x02)

**请求：**
```
AA 02 00 [CRC] 55
```

**响应：**
```
AA 02 04 [4字节温度值float] [CRC] 55
```

#### 5.3.3 设置电流源 (0x10)

**请求：**
```
AA 10 01 [电流源:0=10μA,1=17μA] [CRC] 55
```

**响应：**
```
AA 10 01 [状态:0=成功] [CRC] 55
```

#### 5.3.4 加载分度表 (0x40)

**请求：**（分包发送）
```
AA 40 [LEN] [包序号] [分度表数据...] [CRC] 55
```

**响应：**
```
AA 40 01 [状态:0=成功] [CRC] 55
```

3. 点击"获取设备ID"
4. 发送 GET_DEVICE_ID 命令
5. 接收并显示设备ID
6. 连接成功
```

#### 7.4.2 参数设置流程

2. 输入调整值（如需要）
3. 点击"电流源调整"按钮
4. 发送 SET_CURRENT_SRC 命令
5. 点击"存储参数"按钮
6. 发送 SAVE_PARAM 命令
7. 设置完成
```

## 八、数据存储设计

### 8.1 Flash存储分配

| 地址范围 | 大小 | 用途 |
|----------|------|------|
| 0x08000000 - 0x0803FFFF | 256KB | 程序代码 |
| 0x08040000 - 0x0805FFFF | 128KB | 分度表存储 |
| 0x08060000 - 0x0807FFFF | 128KB | 用户参数 |

### 8.2 参数结构体

```c
// 用户参数结构体
typedef struct {
    uint32_t magic;             // 魔数 0x544D5032 ("TMP2")
    uint8_t current_source;     // 电流源选择 (0:10μA, 1:17μA)
    float current_adj_10uA;     // 10μA调整值
    float current_adj_17uA;     // 17μA调整值
    float temp_4mA;             // 4mA对应温度
    float temp_20mA;            // 20mA对应温度
    uint16_t crc;               // CRC校验
} UserParam_t;
```

### 8.3 分度表格式

```c
// 分度表结构
typedef struct {
    uint32_t magic;             // 魔数 0x54424C00 ("TBL")
    uint16_t point_count;       // 数据点数 (最大4871)
    float voltage[4871];        // 电压值数组 (mV)
    float temperature[4871];    // 温度值数组 (K)
    uint16_t crc;               // CRC校验
} TempTable_t;
```
